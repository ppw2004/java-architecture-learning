## 项目分类

### 练习项目（Practice）
- **目的**：巩固单个知识点
- **规模**：代码量 200-500行
- **时长**：1-3天完成
- **示例**：
  - 集合框架实现简单的学生管理系统
  - IO实现文件复制工具
  - 多线程实现生产者消费者模型

### 综合项目（Integrated）
- **目的**：整合多个知识点，模拟真实项目
- **规模**：代码量 2000-10000行
- **时长**：1-4周完成
- **示例**：
  - 博客系统（CRUD + 用户认证）
  - 电商系统（商品管理 + 订单 + 支付）
  - 秒杀系统（高并发 + 缓存 + 消息队列）

## 项目标准

每个项目应包含：

### 1. 项目文档（docs/）
- **README.md**：项目简介、功能、技术栈、运行方式
- **DESIGN.md**：架构设计、数据库设计
- **API.md**：接口文档
- **CHANGELOG.md**：更新记录

### 2. 源代码规范
- **分层架构**：Controller → Service → Repository
- **命名规范**：类名、方法名、变量名见名知意
- **注释完整**：类注释、方法注释、关键逻辑注释
- **异常处理**：统一的异常处理机制
- **日志记录**：关键操作有日志

### 3. 测试代码
- 单元测试覆盖率 > 70%
- 集成测试
- 性能测试（重要功能）

### 4. 配置文件
- application.yml / application.properties
- pom.xml / build.gradle
- Dockerfile（可选）
